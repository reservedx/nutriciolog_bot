# Product Architecture

## Текущая схема

Проект теперь работает как единая платформа:

- Telegram-бот для быстрого ввода и общения
- веб-кабинет для просмотра истории, прогресса и ручного управления
- одна база данных для обоих интерфейсов
- общий сервисный слой для AI-функций и бизнес-логики

```text
Telegram Bot
   |
   | uses
   v
Shared Services
   |- app/database.js
   |- app/services/nutrition.js
   |
   v
SQLite database on Railway Volume
   ^
   |
Web API + Web Cabinet
```

## Что уже общее

- профиль пользователя
- дневник питания
- журнал веса
- журнал замеров
- прогресс
- AI-вопросы по питанию
- генерация меню на день

## Почему это нормальная продуктовая архитектура

- бот и сайт не дублируют бизнес-логику
- база единая, поэтому данные синхронны
- веб-кабинет можно развивать независимо от бота
- AI-логику можно потом вынести в отдельный backend без переписывания продукта с нуля
